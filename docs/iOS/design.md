# iOS Design

The app uses a native two-column `NavigationSplitView`. On iPad it can show the global recent-sessions list beside the active conversation; on iPhone it becomes a familiar push-navigation flow.

Each project has a lightweight Files browser limited to Markdown and HTML so reference documents stay easy to scan. Files default to newest-modified first; a native Sort menu lets the user switch to alphabetical name or largest-size-first ordering and remembers the selection. Previews support UTF-8 documents up to 10 MB. Markdown opens in a scrollable native preview with typographic headings, inline emphasis, links, lists, task items, quotes, rules, fenced code blocks, and formatted tables. Tables emphasize headers, honor declared column alignment, wrap long cells, alternate row shading, draw cell borders, and scroll horizontally when wider than the device. HTML opens in an embedded browser view. Source code is intentionally absent from the browser, but source links in a conversation or document open a selectable monospaced preview with aligned line numbers and horizontal scrolling so changed files remain inspectable without cluttering the document list. Links to other supported project files open inside the app, and ordinary web links open through the system browser.

The main screen combines sessions from every project, places pinned sessions first, sorts within pinned and unpinned groups by latest host activity, and caps the result at 50. Each row shows the session title, project name, readable activity date and time, status badges for running, failed, unread, or read, and an orange pin for pinned sessions. Leading swipe and long-press actions expose Mark as Read or Unread plus Pin or Unpin. Trailing swipe and long-press actions expose Rename and Delete; deletion requires destructive confirmation and is unavailable while a session is running. The destructive button is always the compact label “Delete,” while the session title appears in the explanatory dialog message. The rename editor trims the title, enforces a 120-character maximum, and preserves activity ordering. The global new-session action asks for a project only when needed; the conversation toolbar's compose action immediately creates and opens a new session in the current project. There is no persistent project switcher. Connection health remains visible in the main and conversation toolbars. On iPhone, the conversation uses a custom Back control with a blue dot whenever any session is unread, making pending activity visible without a project menu. A horizontal right swipe across the conversation invokes the same animated return to the global session list while ignoring short or primarily vertical transcript drags. The app icon badge shows the total unread-session count and clears as sessions are read. Marking a read session unread increments the badge without changing its activity timestamp or list position. Presenting an unread conversation while the app is active marks it read through the host API. A conversation opens at its bottom-most activity so the latest message, running state, or queued prompt is immediately visible. A newly submitted prompt appears optimistically as the blue user bubble before the accepted turn reveals its green working surface; a rejected send removes the optimistic bubble. While an agent is working, the green progress surface shows its latest concise reasoning summary; later summaries replace it rather than creating transcript entries, and the surface disappears when the final result arrives. The pinned composer remains enabled and changes to a queue action. Host-owned queued prompts appear in FIFO order above the composer with pencil and remove controls; the native editor permits multi-line changes until the Mac starts that item. Composer drafts persist locally, while queued prompts persist with their Mac session and continue executing when iOS is suspended or terminated.

A compact project-command bar sits above the composer. Its hammer split control shows and runs the current target from its primary action and opens a target picker from the arrow; selection persists with the host session. One Commit & Push button invokes the same Mac-host workflow as the desktop app. Starting an action adds and scrolls to a green running chat row, which becomes a success or failure row in place. Tapping any project-command row presents a dismissible output screen with live status, the command and project path, exit status, and selectable monospaced output fetched from the Mac.

Connection setup is a sheet led by a prominent Current Status section with a semantic icon and color, plain-language state, saved Mac endpoint, negotiated API version or failure detail, and an inline retry action when the saved Mac is unavailable. The editable setup below it retains the hostname/IP field, numeric port, privacy-sensitive secure token field, validation, and explicit connection test. Repeated startup, manual, and foreground connection requests share one attempt, while transient local-network recovery waits briefly for connectivity before retrying safe reads. Disconnecting keeps the sheet visible so the status change is immediately apparent. System colors, Dynamic Type, SF Symbols, semantic controls, and standard accessibility labels are used throughout.

Appearance can follow the iOS system setting or be pinned to Light or Dark from the connection sheet. The preference applies immediately across navigation, conversations, Markdown, source code, and restricted HTML previews, and persists across launches.
