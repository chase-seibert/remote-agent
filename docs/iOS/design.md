# iOS Design

The app uses a native two-column `NavigationSplitView`. On iPad it can show the global recent-sessions list beside the active conversation; on iPhone it becomes a familiar push-navigation flow.

Each project has a lightweight Files browser limited to Markdown and HTML so reference documents stay easy to scan. Markdown opens in a scrollable native preview with typographic headings, inline emphasis, links, lists, task items, quotes, rules, and fenced code blocks. HTML opens in an embedded browser view. Source code is intentionally absent from the browser, but source links in a conversation or document open a selectable monospaced preview with aligned line numbers and horizontal scrolling so changed files remain inspectable without cluttering the document list. Links to other supported project files open inside the app, and ordinary web links open through the system browser.

The main screen combines sessions from every project, places pinned sessions first, sorts within pinned and unpinned groups by latest host activity, and caps the result at 50. Each row shows the session title, project name, readable activity date and time, status badges for running, failed, unread, or read, and an orange pin for pinned sessions. Leading swipe and long-press actions expose Pin or Unpin. Trailing swipe and long-press actions expose Rename and Delete; deletion requires destructive confirmation and is unavailable while a session is running. The rename editor trims the title, enforces a 120-character maximum, and preserves activity ordering. The global new-session action asks for a project only when needed; the conversation toolbar's compose action immediately creates and opens a new session in the current project. There is no persistent project switcher. Connection health remains visible in the main and conversation toolbars. On iPhone, the conversation uses a custom Back control with a blue dot whenever any session is unread, making pending activity visible without a project menu. A horizontal right swipe across the conversation invokes the same animated return to the global session list while ignoring short or primarily vertical transcript drags. The app icon badge shows the total unread-session count and clears when no unread sessions remain. Presenting an unread conversation while the app is active marks it read through the host API. A conversation opens at its bottom-most activity so the latest message, running state, or queued prompt is immediately visible. While an agent is working, the pinned composer remains enabled and changes to a queue action. Queued prompts appear in FIFO order above the composer, show delivery progress, and can be removed before delivery. Drafts and queued prompts persist across navigation and relaunches.

Connection setup is a sheet with a hostname/IP field, numeric port, privacy-sensitive secure token field, validation, and an explicit connection test. System colors, Dynamic Type, SF Symbols, semantic controls, and standard accessibility labels are used throughout.

Appearance can follow the iOS system setting or be pinned to Light or Dark from the connection sheet. The preference applies immediately across navigation, conversations, Markdown, source code, and restricted HTML previews, and persists across launches.
